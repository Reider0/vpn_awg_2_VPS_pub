
# 🛡️ Dual-Node VPN Shadow System (RU + DE)

Инновационная распределенная система VPN на базе **AmneziaWG**, спроектированная для обхода сложных систем блокировок (DPI) и обеспечения максимальной безопасности. Система разделена на два узла: **Control Plane (РФ)** и **Data Plane (Германия)**.

## 🌟 Основные особенности

  * **Стелс-маршрутизация (гибрид):** Весь зарубежный трафик прозрачно уходит через туннель в Германию (для зарубежных сервисов пользователь выглядит как «из Германии»), а российский трафик (Госуслуги, банки) идёт напрямую через локальный интерфейс РФ. Поверх этого работает слой **antifilter**: ресурсы из реестра РКН, размещённые на российских IP, принудительно уходят через Германию — так обход блокировок не зависит от того, где физически хостится ресурс.
  * **Инвертированное соединение:** Сервер в Германии выступает в роли клиента для сервера в РФ, что маскирует VPN-туннель под обычное входящее соединение из-за рубежа.
  * **Anti-Sharing 2.0:** Умная система грейлистинга IP-адресов. Запоминает доверенные сети пользователя и блокирует ключ при попытке одновременного использования с разных устройств (детектор "перетягивания каната").
  * **Полная изоляция:** Telegram-бот работает в сетевом пространстве WireGuard и обращается к API Telegram через туннель, обходя блокировки РКН без прокси.
  * **DevOps Ready:** Автоматическое обновление обоих серверов из GitHub по одной кнопке в Telegram-боте.

## 🧭 Умная маршрутизация и списки IP

Решение «куда отправить пакет» принимается в ядре через `ipset` + `iptables` (mangle / fwmark 200). Используются два набора, которые обновляются автоматически **раз в 12 часов** фоновым демоном внутри контейнера `ru_wireguard`:

| Набор | Источник | Назначение |
| :--- | :--- | :--- |
| **`ru_nets`** | ipdeny.com (аллокации RIPE) → fallback на резервный список | Российские IP. Трафик в эти сети идёт **напрямую** (банки, госуслуги, локальные сервисы). |
| **`blocked_nets`** | antifilter.download (`allyouneed.lst` = ipsum + subnet) | Реестр блокировок РКН. Если заблокированный ресурс сидит на **российском** IP — он принудительно уходит **через Германию**. |

**Логика на пакет:** российский адрес → напрямую; всё остальное → Германия; заблокированный РКН ресурс → Германия в любом случае.

**Надёжность источников:**

  * Скачанные списки кэшируются на диск (`volumes/wireguard/cache/`). При недоступности сети список **не обнуляется** — используется последняя удачная версия из кэша.
  * При старте контейнера наборы наполняются из кэша мгновенно (тёплый старт), не дожидаясь первого сетевого обновления.
  * Принудительный обход блокировок реализован **нефатально**: даже если набор `blocked_nets` недоступен, узел поднимется на базовой гео-логике, а не уйдёт в полный отказ.

> ⚠️ Важно про antifilter: он отдаёт **только реестр заблокированных РКН ресурсов**, а не список IP России. Поэтому он не заменяет гео-IP, а дополняет его. Заменять гео-IP на antifilter нельзя — иначе весь обычный зарубежный трафик пойдёт напрямую с российского IP, и сервисы перестанут видеть пользователя «из Германии».

## 🏗️ Архитектура системы

### 🇷🇺 VPS\_RU (Мастер-узел)

  * **Ядро:** AmneziaWG Server.
  * **Мозг:** Telegram-бот на Python (python-telegram-bot).
  * **Память:** PostgreSQL 15.
  * **Функции:** Управление пользователями, база данных, умная маршрутизация (`ipset`), прием платежей/тикетов.

### 🇩🇪 VPS\_DE (Агент-узел)

  * **Ядро:** AmneziaWG Client.
  * **Агент:** FastAPI Monitor API.
  * **Функции:** Выход в глобальный интернет, сбор системных логов и метрик Германии для передачи в РФ.

## 🚀 Быстрый старт

### Шаг 1: Подготовка РФ-сервера

1.  Склонируйте репозиторий в `/root/vpn_project`.
2.  Настройте `VPS_RU/.env`:
    ```env
    BOT_TOKEN=ваш_токен
    ADMIN_ID=ваш_id
    GIT_REPO=ссылка_на_гит
    GIT_USERNAME=логин
    GIT_TOKEN=токен_доступа
    POSTGRES_USER=vpn_admin
    POSTGRES_PASSWORD=ваш_надежный_пароль
    POSTGRES_DB=vpndb
    ```
3.  Запустите установку:
    ```bash
    cd VPS_RU && sudo bash install.sh
    ```
4.  В конце установки скрипт выведет содержимое файла `DE_AGENT_CONFIG.txt`. **Скопируйте этот текст**.

### Шаг 2: Подготовка сервера в Германии

1.  Загрузите папку `VPS_DE` на немецкий сервер.
2.  Настройте `VPS_DE/.env` (нужен только для автообновлений).
3.  Запустите установку:
    ```bash
    cd VPS_DE && sudo bash install.sh
    ```
4.  Вставьте скопированный текст в конфиг:
    ```bash
    nano volumes/wireguard/wg0.conf
    # Вставьте текст и сохраните (Ctrl+O, Enter, Ctrl+X)
    ```
5.  Перезапустите агента:
    ```bash
    docker restart de_vpn_agent
    ```

## 🔄 Обновление в проде

Код обоих узлов обновляется из GitHub по кнопке в боте (**Обновить RU / DE**) без ручных шагов:

1.  Закоммитьте и запушьте изменения в `origin/main`.
2.  Нажмите в боте **«Обновить RU»**. Сработает цепочка: флаг → `host_updater.sh` → `deploy.sh` (`git reset --hard origin/main` + `docker compose up -d --build`).
3.  Контейнеры пересоберутся, `ipset`-наборы наполнятся сами, правила `iptables` применятся при старте.

**Что переживает обновление (не теряется):**

  * **База данных** — том `volumes/database`. Схема (`db/init.sql`) применяется только на пустом томе, существующие данные не трогаются.
  * **Ключи WireGuard** — `volumes/wireguard/wg0.conf` и ключи сервера. Идентичность пира определяется PublicKey, а не именем.
  * **`.env`** — восстанавливается из репозитория (он закоммичен). Деплой его не меняет.

## 🛡️ Безопасность и Мониторинг

### Автоматическая защита

  * **SSH Protection:** Скрипт установки предлагает сменить порт SSH и настраивает `ufw`.
  * **Fail2Ban:** Автоматически блокирует IP при попытке подбора паролей.
  * **Kernel Hardening:** Настроены параметры `sysctl` для защиты от SYN-флуда и спуфинга.

### Мониторинг в боте

  * **Дашборд:** Выводит загрузку CPU/RAM/Disk для обоих серверов одновременно.
  * **Алерты:** Бот пришлет уведомление, если на любом из серверов нагрузка превысит 90% или упадёт туннель.
  * **Аудит:** Глубокая диагностика всех систем (сеть, докер, база) одной кнопкой.

## 🛠️ Управление системой

| Команда в боте | Описание |
| :--- | :--- |
| **Reboot RU / DE** | Удаленная перезагрузка физических серверов. |
| **Логи DE** | Чтение системного журнала из Германии прямо в чат. |
| **Бэкап RU (Full)** | Дамп всей БД и всех сгенерированных ключей клиентов. |
| **Бэкап DE** | Сохранение конфигурации туннеля из Германии. |
| **Обновить RU / DE** | Запуск `git pull` и пересборка контейнеров на выбранной ноде. |
| **Сбросить сети** | Очистка истории IP-адресов конкретного пользователя. |

### 🔤 Имена ключей

Имя ключа автоматически приводится к виду, который принимает приложение **AmneziaWG** (иначе оно ругается «неверно задано поле имя»):

  * кириллица транслитерируется в латиницу (`Иван iPhone` → `Ivan_iPhone`, `Алёна` → `Alena`);
  * эмодзи и спецсимволы отбрасываются;
  * длина ограничивается 15 символами (лимит имени тоннеля).

Имя — косметическое: на работу ключа и привязку к Telegram оно не влияет. Существующие ключи не меняются; приведение применяется только к новым именам при создании.

## 📋 Требования (боевые)

  * **РФ:** Ubuntu 22.04+, 1–2 ГБ ОЗУ (2 ГБ комфортно для Postgres + контейнеров). Списки маршрутизации добавляют лишь несколько МБ RAM, обновление списков — фоновая задача раз в 12 часов с минимальной нагрузкой.
  * **Германия:** Ubuntu 22.04+, минимум 1 ГБ ОЗУ (скрипт автоматически создаст SWAP на 2 ГБ для стабильности).
